1 What is promptfoo & what is an assertion?

Promptfoo is an open-source tool for testing prompts and LLMs — like unit tests, but for AI output. You define test cases, run them against one or more models, and promptfoo scores each output.

An assertion is a single check applied to the model's output. There are two families:

FamilyHow it judgesExamples
deterministic Pure code — fast, free, repeatable. No extra model call. contains, regex, is-json, javascript
model-graded Another model (or embeddings) judges quality/meaning. llm-rubric, similar, g-eval, factuality

A test case passes only when all of its assertions pass.

2 Prerequisites

  • Node.js 18+ — check with node --version. No global install needed; we use npx.
  • Python 3.9+ — powers the custom Python grader. Install deps once with the command below.
  • An API key — OpenAI by default (Anthropic works too).
pip install -r requirements.txt
Everything for this part lives in this folder (part1_assertions/). The HTML you're reading is guide.html, next to promptfooconfig.yaml, prompts/assistant.txt, assertions/readability.py, and requirements.txt.

3 Set your API key

Export it in your terminal (simplest), or copy .env.example to .env and fill it in.

export OPENAI_API_KEY="sk-...your-key..."
Running an eval makes real API calls and costs a few cents. The config caps each call at $0.01 via a cost assertion.

4 Create the prompt

Prompts use {{variable}} placeholders that get filled from each test case. This is prompts/assistant.txt:

You are a concise, accurate assistant.
Answer the user's question clearly and factually. Do not add extra commentary.

Question: {{question}}

5 Write promptfooconfig.yaml

This is the heart of promptfoo. Three required sections plus an optional defaultTest:

SectionMeaning
promptsThe prompt template(s) to evaluate.
providersWhich model(s) run the prompts.
testsThe test cases: vars + assert checks.
defaultTestAssertions applied to every test (e.g. latency, cost).

Skeleton:

description: "Promptfoo Assertions Overview — hands-on tutorial"

prompts:
  - file://prompts/assistant.txt

providers:
  - openai:gpt-4o-mini

defaultTest:
  assert:
    - type: latency
      threshold: 10000   # fail if a call is slower than 10s
    - type: cost
      threshold: 0.01    # fail if a call costs more than $0.01

tests:
  # ... test cases go here (next two steps) ...
To use Anthropic instead, swap the provider line for anthropic:messages:claude-haiku-4-5-20251001 and set ANTHROPIC_API_KEY.

6 Add deterministic assertions

These run in code — instant and free. Add these entries under tests:.

# contains / not-contains
- description: "substring is present"
  vars:
    question: "What is the capital of France? Answer in one word."
  assert:
    - type: icontains      # case-insensitive substring
      value: paris
    - type: not-contains   # must NOT appear
      value: London

# regex — match a pattern
- description: "output matches a pattern"
  vars:
    question: "Reply with exactly the word: APPROVED"
  assert:
    - type: regex
      value: "^\\s*APPROVED\\s*$"

# is-json — output parses as JSON
- description: "output is valid JSON"
  vars:
    question: 'Return ONLY a JSON object with keys "city" and "country" for Tokyo.'
  assert:
    - type: is-json

# javascript / python — your own logic; `output` is the model text
- description: "custom JS check"
  vars:
    question: "Give me a one-sentence fun fact about octopuses."
  assert:
    - type: javascript
      value: "output.length > 10 && output.length < 300"

7 Add model-graded assertions

These ask a model (or embeddings) to judge meaning and quality — for when there's no single “correct” string. They use the same provider as the judge.

# llm-rubric — an LLM grades against your rubric
- description: "apology is polite"
  vars:
    question: "A customer is angry their order is late. Write a short apology."
  assert:
    - type: llm-rubric
      value: "The response is apologetic, polite, and offers to help."

# similar — semantic (embedding) similarity to a reference answer
- description: "meaning matches reference"
  vars:
    question: "In one sentence, what is photosynthesis?"
  assert:
    - type: similar
      value: "Photosynthesis is how plants convert sunlight into energy."
      threshold: 0.7

# factuality — output agrees with a known fact
- description: "answer is factually correct"
  vars:
    question: "Boiling point of water at sea level in Celsius?"
  assert:
    - type: factuality
      value: "Water boils at 100 degrees Celsius at sea level."
The complete, ready-to-run version of all this is already saved as promptfooconfig.yaml in this folder — you don't have to retype it.

8 Run the evaluation

From this folder (where promptfooconfig.yaml lives):

npx promptfoo@latest eval

Promptfoo runs every test against the model and prints a pass/fail grid plus a summary like Passed: 9 Failed: 1. The first run downloads promptfoo, so give it a moment.

Re-run only failing tests, or filter while iterating:
npx promptfoo@latest eval --filter-pattern "json"

9 View & read the results

Open the interactive web viewer:

npx promptfoo@latest view

It opens http://localhost:15500 with a table: one row per test, one column per provider. Click any cell to see the full output and exactly which assertions passed or failed, with the grader's reasoning for model-graded ones.

How to read a result:

  • Green / PASS — every assertion in that test passed.
  • Red / FAIL — at least one assertion failed; the reason is shown inline.
  • Score — model-graded asserts return 0.0–1.0; thresholds decide pass/fail.
If a model-graded test occasionally flips pass/fail, that's expected — LLM judges aren't perfectly deterministic. Loosen the rubric or lower the threshold to stabilize.

10 Assertion quick reference

Deterministic

TypePasses when…
equalsOutput exactly equals value.
contains / icontainsOutput includes value (i = case-insensitive).
not-containsOutput does not include value.
regexOutput matches the regex in value.
starts-withOutput begins with value.
is-json / is-sqlOutput is valid JSON / SQL.
javascript / pythonYour code returns true / a passing score.
levenshteinEdit distance to value ≤ threshold.
latencyResponse time (ms) ≤ threshold.
costCall cost ($) ≤ threshold.

Model-graded

TypePasses when…
llm-rubricAn LLM judges the output meets your written rubric.
g-evalChain-of-thought scoring against criteria.
similarEmbedding cosine similarity to value ≥ threshold.
factualityOutput is factually consistent with value.
answer-relevanceOutput is relevant to the original question.
context-faithfulnessOutput stays faithful to provided context (RAG).
